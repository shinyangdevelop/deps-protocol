# DEPS - 프로토콜 익스텐션

DEPS 프로토콜은 각 구현체가 필수적으로 구현해야 하는 기능과 선택적으로 구현할 수 있는 기능을 구분한다. 본 문서는 DEPS 프로토콜의 선택적 기능인 **익스텐션**에 대해 설명한다.

## 1. 익스텐션

익스텐션은 DEPS 프로토콜의 선택적 기능이다. 익스텐션은 **공식 익스텐션**과 **비공식 익스텐션**으로 구분된다.

- 공식 익스텐션은 DEPS가 공식적으로 정의한 익스텐션이다. **익스텐션 문서**를 통해 정의된다.
- 비공식 익스텐션은 DEPS의 구현체가 자체적으로 정의한 익스텐션이다.

> [!NOTE]
> 공식이든 비공식이든 익스텐션은 이 레포지터리에 기록해서 표준화하는 것을 권장한다. 수많은 익스텐션이 수많은 곳에 흩어져 있다면 개발하는 쪽에선 다소 슬퍼진다.

각 구현체는 익스텐션을 구현하지 않아도 기본 상호작용에서 다른 구현체와 호환될 수 있어야 한다.

### 1.1. 익스텐션 식별자와 익스텐션 버전 식별자

익스텐션은 고유한 **익스텐션 식별자**를 가진다. 익스텐션 식별자는 아래와 같은 형식을 따른다.

```
공식:   deps.[익스텐션 이름]
비공식: [네임스페이스].[익스텐션 이름]

deps.client-interaction
nulta.my-cool-extension
somegroup.someproject.someextension
```

- 익스텐션 식별자는 겹치지 않게 노력해야 한다.
- 익스텐션 식별자는 항상 소문자로 쓴다.
- [익스텐션 이름]에는 `.`이 들어가지 않는다. `.`이 들어갔다면 그 부분은 네임스페이스이다.
- [네임스페이스]를 결정하는 규칙은 없지만, 서로 겹치지 않도록 노력해야 한다.

**익스텐션 버전 식별자**는 익스텐션 식별자에 버전 정보를 추가한 것이다. 익스텐션 버전 식별자는 아래와 같은 형식을 따른다.

```
[익스텐션 식별자]@[메이저 버전].[마이너 버전]

deps.client-interaction@1.0
deps.client-interaction@2.7
nulta.my-cool-extension@0.0
```

- [메이저 버전]이 다른 익스텐션간의 상호작용은 호환되지 않는다.
- 만약 서버가 여러 [메이저 버전]으로의 상호작용을 지원한다면, 해당하는 메이저 버전을 모두 표시해야 한다.
- [마이너 버전]이 다른 익스텐션은 스펙은 약간 달라도 상호 호환이 가능하다.

## 2. 익스텐션이 할 수 있는 것

익스텐션은 다음과 같은 일을 할 수 있다.

- 추가적인 API 엔드포인트를 정의할 수 있다.
- 이미 존재하는 API 요청이나 응답에 추가적인 필드를 정의할 수 있다.
- 상대 구현체의 익스텐션 지원 여부를 확인한 뒤, 합의된 방식으로 자유롭게 상호작용할 수 있다.

익스텐션이 다음과 같은 일을 해서는 안 된다.

- 기존 API 엔드포인트를 제거하거나 의미를 변경해서는 안 된다.
- 기존 API 엔드포인트의 필드를 제거하거나 의미를 변경해서는 안 된다.

이는 익스텐션의 구현 유무와 상관없이 기본 상호작용이 호환될 수 있도록 하기 위함이다.

### 2.2. 필드의 변경 가능성

익스텐션의 존재로 인해, 모든 API 요청이나 응답에는 정의되지 않은 임의의 필드가 추가적으로 포함될 수 있다.

따라서 각 구현체는 이러한 가능성을 염두에 두고 적절히 처리할 수 있어야 한다. 이를테면 구현체는 정의되지 않은 필드를 오류 없이 무시할 수 있어야 한다.
