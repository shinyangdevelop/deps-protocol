*Decentralized Problem Solving :: Rev 0.1*

읽어주셔서 감사합니다. 여기서 `배틀`을 `문제풀이`로, `체육관`을 `문제서버`로 바꾸면 **DEPS Protocol**이 됩니다.

## 1. 개요

DEPS 프로토콜은 Ed25519 암호체계를 기반으로 설계된 분산형 문제풀이 서버들의 연합 프로토콜이다.

이 프로토콜의 특징 및 설계 목적은 다음과 같다.

1. 사용자는 플랫폼의 존속과 관계없이 자신의 계정과 문제 풀이 기록에 대한 완전한 소유권을 가진다.
2. 작은 서비스 여러 개의 연합체계를 통해 생태계 전체의 안정성을 강화한다.
3. 사용자가 자신의 실력을 간편하게 확인하고 증명할 수 있는 시스템을 제공한다.

## 2. 구성

DEPS 생태계는 크게 4종류의 주체로 구성된다.

### 2.1. 아이덴티티 (Identity)

**아이덴티티**는 사용자를 정의하는 최상위 객체이다. 아이덴티티는 특정 서버에 종속되지 않는다.

아이덴티티는 닉네임 등의 기본 정보와 **아이덴티티 식별자**를 가진다. 아이덴티티 식별자는 Base64로 인코딩한 Ed25519 공개 키이며, 한 번 생성되면 절대 변경되지 않는다. 이 값은 고유한 식별자 역할을 수행함과 동시에 사용자의 신원을 인증하는 데에도 사용된다.

### 2.2. 홈서버 (Home Server)

> [!NOTE]
> 홈서버 없이 아이덴티티를 직접 관리하는 것도 가능하다. 하지만 홈서버를 두는 것이 훨씬 편리하다.
> 
> 아래는 홈서버의 가장 기본적인 역할만 나열한 것이다. 문제 조회 및 제출 UI까지 홈서버가 제공하도록 구현할 수도 있다.

**홈서버**는 사용자의 아이덴티티와 데이터를 대신 보관해주는 서버이다.

- 사용자가 획득한 각종 증명(문제증표 및 실력증표)을 저장하고 관리한다.
- 사용자를 쉽게 나타내는 별칭인 **핸들네임**을 부여할 수 있다.
- 사용자는 언제든 자신의 데이터를 추출하여 다른 홈서버로 옮길 수 있다.

### 2.3. 문제서버 (Problem Server)

> [!NOTE]
> **문제는 각 문제서버의 운영 주체가 자율적으로 제작한다.** 또한, 문제 채점을 어떻게 하는지는 프로토콜에서 정의하지 않으며 전적으로 문제서버의 구현에 따른다.

**문제서버**는 사용자에게 여러 문제를 제공하고, 제출받은 답안을 채점하는 서버이다.

- 정답을 맞춘 아이덴티티에게는 문제서버의 키로 서명된 **문제증표** 를 발급한다.
